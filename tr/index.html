Tarihçe

In February 1999, Tom Christiansen announced the Perl Power Tools project to provide a unified BSD toolbox. Perl is the same (mostly) everywhere you go and the same programs could run the same everywhere instead of being reimplemented for each platform. The Perl versions would have the same switches and features.

After that, the tools were lost in a minor Dark Ages, after which Casey West and Sean Dague resurrected the project in 2001. They made it onto CPAN as "ppt", was hosted at SourceForge, had a mailing list at perl.ppt, and a website at ppt.perl.org (now dead, but the source is in the old repo).

However, the project languished once again in an environment without easy forking and pull requests.

In 2014, brian d foy revitalized the project, changing the name to PerlPowerTools to make it much easier to search and discover. He moved it to GitHub and created its website www.perlpowertools.com with GitHub Pages.

Future

Now it's time to bring Perl Power Tools up to current standards, including tests. With a test suite, not only can we ensure correct behavior but we can provide a reference implementation should someone want to to the same in some other language.
